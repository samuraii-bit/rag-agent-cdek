# Тестовое задание для стажера LLM Engineer 

## 📋 Название: Разработка контекстного RAG-агента на LangGraph

### 1. Цель задания

Реализовать сервис чат-бота, который консультирует пользователей по правилам международной стажировки.  
**Ключевая особенность:** Бот должен уметь поддерживать диалог (контекст) и **задавать уточняющие вопросы**, если запрос неоднозначен (например, не указана страна). Бот не должен «выдумывать» ответы (галлюцинировать) — только строгий ответ по базе знаний.

Результат должен быть в виде ссылки на Git-репозиторий (публичный или с приглашением пользователя `ihomyak`)

### 2. Технический стек

Git
Docker Compose
Python - FastAPI, LangGraph
База знаний - 5 текстовых файлов
LLM - возможность подключить любую LLM (хоть локальную, хоть OPenAI )

### 3. Входные данные (База Знаний)

Вам необходимо создать папку `data/` и разместить в ней следующие 5 файлов.  
**Важно:** Документы `germany_rules.txt` и `france_rules.txt` намеренно похожи по структуре, но содержат разные данные. Это ключевой кейс для проверки логики уточнения.

#### 📄 `data/general_info.txt`

```text

Для участия необходимо заполнить анкету на сайте и приложить резюме.
Отбор проходит в два этапа: тестирование и собеседование.
Язык общения в программе — английский.
```

#### 📄 `data/deadlines.txt`

```text

Дедлайн подачи документов — 25 апреля.

Начало стажировки — 1 июня.
```

#### 📄 `data/benefits.txt`

```text

Выдается страховка на весь период.
По окончании выдается сертификат.
```

### 4. Функциональные требования

#### 4.1. Работа с Git

* Репозиторий должен быть инициализирован
* В репо не должно быть секретов (API ключей)

#### 4.2. Контейнеризация (Docker Compose)

Проект должен запускаться командой:

```bash
docker-compose up --build
```

#### 4.3. Описание проекта

В корне проекта должен быть файл readm.mdс описание как запустить и что он выполняет.

#### 4.4. API (FastAPI)

Реализовать **один эндпоинт**:

* **URL:** `POST /chat`
* Content-Type: application/json

#### 4.5. LLM

В проекте должны быть конфигурации и возможность настроить подключение к любой LLM.

Если есть какие-то особенности или ограничения в использовании моделей - это должно быть описано в файле `readme.md`
